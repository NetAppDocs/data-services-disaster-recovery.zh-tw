= Amazon EVS 與NetApp災難復原的先決條件
:allow-uri-read: 


[role="lead"]
在繼續使用NetApp災難復原配置 Amazon EVS 之前，您應該確保滿足幾個先決條件。

具體來說，請執行以下操作：

* 建立 vCenter 使用者帳戶，該帳戶具有NetApp災難復原所需的特定 VMware 權限，以執行必要的操作。
+

NOTE: 我們不建議使用預設的“\administrator@vsphere.com”管理員帳戶。相反，您應該在將參與 DR 過程的所有 vCenter 叢集上建立一個NetApp災難復原特定的使用者帳戶。有關所需特定權限的列表，請參閱link:vcenter-privileges.html["NetApp災難復原所需的 vCenter 權限"]。

* 確保將託管受NetApp災難復原保護的虛擬機器的所有 vCenter 資料儲存區都位於NetApp ONTAP儲存資源上。
+
在NetApp ONTAP上使用Amazon FSx時，此服務支援 iSCSI（而非 FC）上的 NFS 和 VMFS。雖然該服務支援 FC，Amazon FSx for NetApp ONTAP不支援。

* 確保您的 Amazon EVS vCenter 已連線至Amazon FSx for NetApp ONTAP儲存叢集。
* 確保所有受保護的虛擬機器上都安裝了 VMware 工具。
* 確保您的本地網路使用 Amazon 認可的連線方法連接到您的 AWS VPC 網路。我們建議您使用 AWS Direct Connect、AWS Private Link 或 AWS Site-to-Site VPN。
